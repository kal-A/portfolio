# Implementation Steps and Final Quality Check

## Implementation steps

1. Read the existing portfolio codebase.
2. Read existing project data files, case study files, card components, and
   homepage layout.
3. Read any existing `CARD_REWRITE_INSTRUCTIONS.md` or prior instructions,
   but treat this `docs/redesign/` directory as the final source of truth
   where they conflict.
4. Identify current homepage sections.
5. Identify current case study/project card structure.
6. Identify whether project cards support images, metadata, tags, and
   artifact chips.
7. Add or update card variants: visual project card, systems/process project
   card, supporting/archive card. See
   [06-project-card-types.md](06-project-card-types.md).
8. Update the homepage structure: Hero, Selected Case Studies, Design Work /
   Visual Evidence, How I Work, Project Archive, Skills / Ways I Work, About,
   Contact. See [03-homepage-and-hero.md](03-homepage-and-hero.md).
9. Update the hero copy to be grounded, specific, and less generic.
10. Update selected case studies based on the priority and artifact reality
    in [02-artifact-reality-and-treatment.md](02-artifact-reality-and-treatment.md)
    and [04-selected-case-studies.md](04-selected-case-studies.md).
11. Make RoomEase, Hera, Greenhouse, and HCI/Pill Pal visually stronger.
12. Make ForceN a public-safe systems/process case study.
13. Keep PathPeer and Informatica in supporting/archive treatment unless
    stronger artifacts are added.
14. Audit all metrics and remove or soften unsupported claims — see
    [08-metrics-audit.md](08-metrics-audit.md).
15. Add confidentiality notes where needed.
16. Make sure design ability is visible for UX/UI and Product Design
    applications.
17. Run the site locally.
18. Check responsiveness and layout.
19. Summarize changes made.
20. List remaining content gaps and artifacts needed.

## Final quality check

Before finishing, answer these questions:

1. Does the portfolio visibly show UX/UI and product design ability?
2. Does Greenhouse clearly show the strongest artifact-backed design work?
3. Does RoomEase feel like a strong product/design/systems case study?
4. Does Hera feel like a strong startup/product design case study?
5. Does HCI/Pill Pal clearly show UX research and design method?
6. Is ForceN handled honestly without fake artifacts?
7. Are PathPeer and Informatica kept appropriately lower-priority?
8. Are any projects being visually exaggerated beyond the artifacts actually
   available?
9. Are any metrics unsupported, inflated, or fake-sounding?
10. Does the site still work for PM, TPM, Product Design, and UX/UI roles?
11. Does the site feel like Kamal rather than a generic AI template?
