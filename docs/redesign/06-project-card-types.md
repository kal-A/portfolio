# Project Card Types

Create or support three main project card variants.

## 1. Visual project card

Use for: **RoomEase, Greenhouse, Hera, HCI / Pill Pal**

Card should include:
- image/mockup/asset preview
- project name
- short problem statement
- role
- timeline or context
- tools
- artifact chips
- 2–3 sentence grounded summary
- CTA

Example artifact chips: Figma · Prototype · Campaign Assets · UX Flow ·
Usability Evaluation · Report · Dataset · Dashboard · Presentation

## 2. Systems/process project card

Use for: **ForceN**, possibly **Informatica**

Card should include:
- simple diagram/process visual instead of screenshot
- project name
- short problem statement
- role
- workflow stages
- artifact chips
- confidentiality note if needed
- CTA

Example artifact chips: Workflow Map · Process Tracker · Handoff Table ·
Documentation · Internal Materials Omitted

## 3. Supporting/archive card

Use for: **PathPeer, Informatica**, smaller school/personal projects.

Concise, honest, no fake visuals, no inflated impact. See
[02-artifact-reality-and-treatment.md](02-artifact-reality-and-treatment.md)
for what these cards should and shouldn't include.

For the exact copy format/tone rules (problem-first framing, banned words,
metric verification), see the root-level
[`CARD_REWRITE_INSTRUCTIONS.md`](../../CARD_REWRITE_INSTRUCTIONS.md) and
[08-metrics-audit.md](08-metrics-audit.md).
