# Visual Style Direction

Use a visual style that feels:

- clean
- editorial
- structured
- thoughtful
- modern
- slightly notebook/archive-inspired
- design-forward where artifacts exist
- systems-minded where process matters

## Suggestions

- subtle grid lines or engineering-paper feel
- project cards that feel like dossiers
- metadata rows
- artifact chips
- strong thumbnails for visual projects
- diagrams for process projects
- tasteful use of whitespace
- sections that feel intentionally structured, not empty
- "constraints" and "what I'd improve next" blocks inside case studies

## Avoid

- bare-bones text-only layout
- generic SaaS landing page feel
- too many buzzwords
- too much empty whitespace
- making every card look the same
- fake dashboards/mockups
- hiding design work under systems language
